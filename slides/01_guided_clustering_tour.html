1. Seurat 객체 준비

count matrix를 읽고 분석을 담을 객체를 만듭니다.

필요한 패키지를 불러옵니다

library(dplyr)
library(Seurat)
library(patchwork)
  • Seurat: 단일세포 데이터 분석 패키지
  • dplyr: 표 형태 결과 정리 패키지
  • patchwork: 여러 plot 조합 패키지

library()와 패키지는 2회차에서 자세히 배웁니다.

count matrix를 읽습니다

pbmc.data <- Read10X(
  data.dir = "data/pbmc3k/filtered_gene_bc_matrices/hg19/"
)

입력: Cell Ranger 출력 폴더
출력: 유전자 × 세포 UMI count matrix

pbmc.data[c("CD3D", "TCL1A", "MS4A1"), 1:5]

Seurat 객체를 만듭니다

pbmc <- CreateSeuratObject(
  counts = pbmc.data,
  project = "pbmc3k",
  min.cells = 3,
  min.features = 200
)
pbmc
  • 원시 count matrix를 pbmc 객체에 저장합니다.
  • 너무 적은 세포에서 검출된 유전자와 유전자가 거의 없는 세포를 1차로 제외합니다.
  • 이후 분석 결과도 같은 객체에 차례로 추가됩니다.

count matrix는 대부분 0입니다

dense.size <- object.size(as.matrix(pbmc.data))
sparse.size <- object.size(pbmc.data)
dense.size / sparse.size
  • 한 세포에서는 전체 유전자 중 일부만 검출됩니다.
  • 0을 모두 저장하면 메모리를 많이 사용합니다.
  • Seurat는 가능한 경우 희소 행렬을 사용합니다.

행렬과 희소 행렬은 6회차에서 다시 다룹니다.

2. QC와 세포 선택

분석에 사용할 품질 좋은 세포를 선택합니다.

세 가지 QC 지표를 확인합니다

QC 지표 의미 지나치게 높거나 낮을 때 의심할 점
nFeature_RNA 검출된 유전자 수 빈 방울, 저품질 세포, doublet
nCount_RNA 전체 UMI 수 RNA가 적은 세포 또는 doublet
percent.mt 미토콘드리아 유전자 비율 손상되거나 죽어가는 세포

QC 기준은 모든 데이터에 동일하게 적용하는 정답이 아닙니다. 분포와 실험 조건을 함께 확인해야 합니다.

미토콘드리아 유전자 비율을 계산합니다

pbmc[["percent.mt"]] <- PercentageFeatureSet(
  pbmc,
  pattern = "^MT-"
)
  • 이름이 MT-로 시작하는 유전자 count의 비율을 계산합니다.
  • 결과는 각 세포의 metadata에 percent.mt 열로 추가됩니다.
head(pbmc@meta.data, 5)

QC 지표의 분포를 시각화합니다

VlnPlot(
  pbmc,
  features = c("nFeature_RNA", "nCount_RNA", "percent.mt"),
  ncol = 3
)

플롯 보기

확인할 것

  • 대부분의 세포가 모여 있는 범위
  • 극단적으로 유전자 또는 UMI 수가 많은 세포
  • 미토콘드리아 비율이 높은 세포

QC 지표 사이의 관계도 확인합니다

plot1 <- FeatureScatter(
  pbmc, feature1 = "nCount_RNA", feature2 = "percent.mt"
)
plot2 <- FeatureScatter(
  pbmc, feature1 = "nCount_RNA", feature2 = "nFeature_RNA"
)
plot1 + plot2

플롯 보기

산점도를 이용하면 한 지표만 볼 때 놓칠 수 있는 세포를 확인할 수 있습니다.

QC 조건으로 세포를 선택합니다

pbmc <- subset(
  pbmc,
  subset = nFeature_RNA > 200 &
    nFeature_RNA < 2500 &
    percent.mt < 5
)
  • 검출 유전자가 너무 적거나 많은 세포를 제외합니다.
  • 미토콘드리아 비율이 5% 이상인 세포를 제외합니다.

>, <, &와 필터 조건은 8회차에서 자세히 배웁니다.

3. 표준 전처리

세포를 비교하고 주요 생물학적 변이를 찾을 준비를 합니다.

세포 간 발현량을 비교할 수 있게 정규화합니다

pbmc <- NormalizeData(
  pbmc,
  normalization.method = "LogNormalize",
  scale.factor = 10000
)

LogNormalize는 각 세포의 발현량을 전체 count로 나누고, 일정한 크기로 맞춘 뒤 log 변환합니다.

오늘은 수식보다 “세포 간 비교를 준비한다”는 목적에 집중합니다.

세포 간 차이를 잘 보여주는 유전자를 선택합니다

pbmc <- FindVariableFeatures(
  pbmc,
  selection.method = "vst",
  nfeatures = 2000
)

top10 <- head(VariableFeatures(pbmc), 10)
  • 세포마다 발현량 차이가 큰 유전자 2,000개를 선택합니다.
  • 이후 PCA는 주로 이 유전자들의 정보를 사용합니다.

고변이 유전자를 시각화합니다

plot1 <- VariableFeaturePlot(pbmc)
plot2 <- LabelPoints(
  plot = plot1,
  points = top10,
  repel = TRUE
)
plot1 + plot2

플롯 보기

점 하나는 유전자 하나를 나타냅니다. 라벨은 변이가 큰 대표 유전자입니다.

유전자별 값의 크기를 맞춥니다

all.genes <- rownames(pbmc)
pbmc <- ScaleData(pbmc, features = all.genes)
  • 유전자별 평균을 0에 가깝게 맞춥니다.
  • 유전자별 분산을 1에 가깝게 맞춥니다.
  • 발현량이 큰 일부 유전자만 PCA를 지배하는 것을 줄입니다.

4. PCA와 차원 선택

수천 개 유전자의 정보를 차원 축소를 통해 더 적은 축으로 요약합니다.

PCA(Principal Component Analysis)를 실행합니다

PCA(주성분 분석)는 많은 변수에 포함된 정보를 소수의 새로운 변수로 요약합니다.

pbmc <- RunPCA(
  pbmc,
  features = VariableFeatures(object = pbmc)
)

입력: 스케일링된 고변이 유전자
출력: 세포의 PC 좌표와 유전자의 PC loading

각 PC는 세포 사이의 주요한 발현 변화 패턴을 요약합니다.

어떤 유전자가 PC를 만드는지 확인합니다

print(pbmc[["pca"]], dims = 1:5, nfeatures = 5)

VizDimLoadings(
  pbmc,
  dims = 1:2,
  reduction = "pca"
)

출력 보기

loading의 절댓값이 큰 유전자는 해당 PC를 구분하는 데 크게 기여합니다.

PCA 공간의 세포와 유전자를 시각화합니다

DimPlot(pbmc, reduction = "pca")

DimHeatmap(
  pbmc,
  dims = 1,
  cells = 500,
  balanced = TRUE
)
  • DimPlot(): PCA 공간에서 세포의 분포 출력 보기
  • DimHeatmap(): PC를 정의하는 유전자와 세포의 패턴 출력 보기

사용할 PC 수를 결정합니다

ElbowPlot(pbmc)

출력 보기

  • PC를 늘릴수록 설명하는 새로운 정보는 점차 줄어듭니다.
  • elbow가 완만해지는 구간과 PC별 유전자를 함께 확인합니다.
  • 공식 PBMC 3K 예제는 이후 분석에 PC 1~10을 사용합니다.

PC 수는 정답 하나로 결정되지 않습니다. 공식 튜토리얼도 여러 PC 수로 결과를 비교하도록 권장합니다.

5. 그래프 기반 클러스터링

PCA 공간에서 비슷한 세포를 연결하고 군집을 찾습니다.

비슷한 세포의 이웃 그래프를 만듭니다

pbmc <- FindNeighbors(pbmc, dims = 1:10)
  • PC 1~10에서 가까운 세포를 찾습니다.
  • 이웃을 많이 공유하는 세포 사이의 연결을 강화합니다.
  • 결과 그래프는 pbmc 객체에 저장됩니다.

연결이 조밀한 세포들을 군집으로 나눕니다

pbmc <- FindClusters(pbmc, resolution = 0.5)

head(Idents(pbmc), 5)
  • resolution이 높을수록 일반적으로 더 세분된 군집이 만들어집니다.
  • 공식 예제에서는 9개 군집(0~8)이 생성됩니다.
  • 군집 번호 자체에는 아직 생물학적 이름이 없습니다.

6. UMAP 시각화

세포 사이의 국소적인 관계를 2차원에서 살펴봅니다.

UMAP을 계산하고 군집을 표시합니다

UMAP(Uniform Manifold Approximation and Projection)은 고차원 데이터를 2차원 또는 3차원 공간으로 축소하여 시각화하는 방법

pbmc <- RunUMAP(pbmc, dims = 1:10)

DimPlot(pbmc, reduction = "umap")

출력 보기

  • 점 하나는 세포 하나입니다.
  • 색은 FindClusters()가 지정한 군집입니다.
  • 가까운 점들은 선택한 특징 공간에서 서로 유사합니다.

UMAP은 고차원 데이터의 완전한 지도가 아닙니다. 군집 사이 거리나 모양만으로 생물학적 결론을 내리지 않습니다.

중간 객체를 저장할 수 있습니다

saveRDS(
  pbmc,
  file = "output/pbmc_tutorial.rds"
)
  • 시간이 오래 걸리는 앞 단계를 다시 실행하지 않아도 됩니다.
  • readRDS()로 같은 객체를 다시 불러올 수 있습니다.
  • 분석 단계와 파일명을 기록하면 재현성이 좋아집니다.

7. 클러스터 마커 탐색

각 군집을 다른 세포들과 구분하는 유전자를 찾습니다.

한 군집의 마커를 찾습니다

cluster2.markers <- FindMarkers(
  pbmc,
  ident.1 = 2
)

head(cluster2.markers, n = 5)

군집 2와 나머지 모든 세포를 비교하여 차등 발현 유전자를 찾습니다.

확인할 열: avg_log2FC, pct.1, pct.2, p_val_adj

지정한 군집끼리 비교할 수도 있습니다

cluster5.markers <- FindMarkers(
  pbmc,
  ident.1 = 5,
  ident.2 = c(0, 3)
)

이 코드는 군집 5를 군집 0과 3의 세포들과 비교합니다.

어떤 집단을 비교하는지에 따라 “마커 유전자”의 의미가 달라집니다.

모든 군집의 양의 마커를 찾습니다

pbmc.markers <- FindAllMarkers(
  pbmc,
  only.pos = TRUE
)

pbmc.markers %>%
  group_by(cluster) %>%
  dplyr::filter(avg_log2FC > 1)

cluster0.markers <- FindMarkers(
  pbmc,
  ident.1 = 0,
  logfc.threshold = 0.25,
  test.use = "roc",
  only.pos = TRUE
)
  • 각 군집을 나머지 모든 세포와 비교합니다.
  • only.pos = TRUE는 해당 군집에서 더 높게 발현되는 유전자만 반환합니다.
  • 공식 튜토리얼은 이어서 ROC 검정을 선택하는 예시도 보여줍니다.

마커 발현을 여러 방식으로 확인합니다 (바이올린 플롯)

VlnPlot(
  pbmc,
  features = c("MS4A1", "CD79A")
)

VlnPlot(
  pbmc,
  features = c("NKG7", "PF4"),
  slot = "counts",
  log = TRUE
)
  • VlnPlot(): 군집별 발현 분포 출력 보기
  • slot = "counts": 정규화된 값 대신 raw count를 표시하는 예시 출력 보기

마커 발현을 여러 방식으로 확인합니다 (피처 플롯)

FeaturePlot(
  pbmc,
  features = c(
    "MS4A1", "GNLY", "CD3E", "CD14", "FCER1A",
    "FCGR3A", "LYZ", "PPBP", "CD8A"
  )
)
  • FeaturePlot(): UMAP 위의 발현 위치 출력 보기

대표 마커를 heatmap으로 비교합니다

pbmc.markers %>%
  group_by(cluster) %>%
  dplyr::filter(avg_log2FC > 1) %>%
  slice_head(n = 10) %>%
  ungroup() -> top10

DoHeatmap(pbmc, features = top10$gene) + NoLegend()

출력 보기

각 군집의 대표 마커가 어느 세포에서 높게 발현되는지 한 번에 비교합니다.

전체 실행 순서를 다시 연결합니다

  1. Read10X() → CreateSeuratObject()
  2. QC 계산·시각화 → subset()
  3. NormalizeData() → FindVariableFeatures() → ScaleData()
  4. RunPCA() → PC 수 결정
  5. FindNeighbors() → FindClusters()
  6. RunUMAP() → DimPlot()
  7. FindMarkers() → FindAllMarkers() → 마커 시각화
  8. RenameIdents() → saveRDS()

다음 코드에서 각 줄이 pbmc 객체에 추가하는 정보를 말해 보세요.
