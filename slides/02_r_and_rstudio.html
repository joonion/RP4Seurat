R과 RStudio, 코드 실행의 기초

Seurat 분석 코드를 실행할 환경 준비하기

2회차

오늘의 학습 목표

  1. 콘솔과 스크립트를 구분하고 R 코드를 실행할 수 있다.
  2. 객체를 만들고 작업 디렉터리와 도움말을 확인할 수 있다.
  3. 분석에 필요한 패키지를 설치하고 불러올 수 있다.

1회차 코드에서 다시 볼 부분

library(Seurat)
pbmc.data <- Read10X(data.dir = "data/pbmc3k/")
pbmc <- CreateSeuratObject(counts = pbmc.data)

오늘은 분석 방법보다 이 코드가 어디에서 실행되고, 결과가 어디에 저장되는지 배웁니다.

R과 RStudio는 역할이 다릅니다

도구 역할
R 코드를 계산하는 언어이자 실행 환경
RStudio 코드를 작성하고 실행하기 편하게 만든 IDE
Quarto 코드·설명·결과를 문서와 슬라이드로 변환

RStudio를 닫아도 R 언어가 사라지는 것은 아닙니다.

콘솔은 한 줄씩 즉시 실행합니다

1 + 2
sqrt(16)
  • 입력한 표현식을 바로 계산합니다.
  • 짧은 확인과 실험에 적합합니다.
  • 이전 명령은 기록에 남지만 분석 코드 관리에는 불편합니다.

콘솔의 >는 입력을 기다리는 표시이며 코드에 직접 입력하지 않습니다.

스크립트는 분석 과정을 저장합니다

# 02_r_basics.R
sample_name <- "pbmc3k"
min_features <- 200

sample_name
min_features
  • 여러 줄을 순서대로 작성합니다.
  • 수정하고 다시 실행할 수 있습니다.
  • 분석 과정을 동료와 공유할 수 있습니다.

주석은 코드의 이유를 기록합니다

# PBMC 3K 분석에서 사용할 최소 검출 유전자 수
min_features <- 200

좋은 주석은 코드가 무엇을 하는지 반복하기보다 왜 필요한지 설명합니다.

# 200을 저장한다                 # 피하기
# 저품질 세포의 1차 제외 기준    # 권장

객체는 값을 붙잡아 두는 이름입니다

min_features <- 200
project_name <- "pbmc3k"
  • <- 오른쪽을 계산합니다.
  • 계산 결과를 왼쪽 이름에 저장합니다.
  • 이후 이름을 사용하면 저장된 값을 다시 가져옵니다.
min_features
project_name

객체 이름은 의미가 드러나게 씁니다

# 권장
min_features <- 200
mitochondrial_cutoff <- 5
pbmc_counts <- 2700

# 의미를 알기 어려움
x <- 200
a <- 5
data1 <- 2700
  • 영문자, 숫자, .와 _를 사용할 수 있습니다.
  • 숫자로 시작할 수 없습니다.
  • R은 대소문자를 구분합니다: pbmc와 PBMC는 다릅니다.

작업 디렉터리는 상대 경로의 출발점입니다

getwd()
list.files()
프로젝트 루트/
├─ data/
├─ tools/
├─ slides/
└─ output/

프로젝트 루트에서 실행하면 data/pbmc3k/ 같은 상대 경로를 일관되게 사용할 수 있습니다.

직접 경로를 바꾸기보다 프로젝트를 사용합니다

# 환경마다 달라지므로 피하기
setwd("C:/Users/name/Desktop/project")

# 프로젝트 루트 기준 상대 경로 권장
counts_path <- "data/pbmc3k/"
  • RStudio Project 또는 Quarto Project를 엽니다.
  • 코드에 개인 컴퓨터의 절대 경로를 남기지 않습니다.
  • Windows 경로는 /를 사용하면 이스케이프 문제를 줄일 수 있습니다.

함수 도움말을 직접 확인합니다

?CreateSeuratObject
help("CreateSeuratObject")
args(CreateSeuratObject)
example(mean)

도움말에서 먼저 볼 항목:

  1. Usage: 함수 호출 형태
  2. Arguments: 인자의 의미
  3. Value: 반환값
  4. Examples: 실행 예제

패키지는 설치와 불러오기가 다릅니다

# 컴퓨터에 한 번 설치
install.packages("Seurat")

# R 세션을 시작할 때 불러오기
library(Seurat)
  • 설치는 패키지 파일을 컴퓨터에 저장합니다.
  • library()는 현재 R 세션에서 함수를 사용할 수 있게 합니다.
  • 강의 중 매번 install.packages()를 실행하지 않습니다.

함수의 소속을 명시할 수도 있습니다

dplyr::filter(marker_table, avg_log2FC > 1)
Seurat::NormalizeData(pbmc)

패키지::함수 표기는 같은 이름의 함수가 여러 패키지에 있을 때 유용합니다.

오류 메시지는 읽을 수 있는 정보입니다

library(Seuart)
Error in library(Seuart) :
  there is no package called 'Seuart'

확인 순서:

  1. 철자가 맞는가?
  2. 패키지가 설치되었는가?
  3. 대소문자가 맞는가?

오늘의 핵심

  • 콘솔은 즉시 실행, 스크립트는 분석 과정 저장에 사용합니다.
  • <-는 계산 결과를 객체에 저장합니다.
  • 상대 경로는 프로젝트 루트를 기준으로 작성합니다.
  • 패키지는 한 번 설치하고 세션마다 불러옵니다.
  • 도움말과 오류 메시지는 분석 과정의 중요한 정보입니다.

  3. 절대 경로보다 상대 경로가 재현성에 유리한 이유는 무엇인가요?
  4. <-의 오른쪽과 왼쪽에는 각각 무엇이 있나요?
