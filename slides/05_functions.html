함수 호출과 인자 읽기

Seurat 함수의 입력과 출력 해석하기

5회차

오늘의 학습 목표

  1. 함수, 인자, 기본값, 반환값의 관계를 설명할 수 있다.
  2. 이름 있는 인자와 중첩된 함수 호출을 읽을 수 있다.
  3. NormalizeData() 등 Seurat 함수의 입력 객체와 주요 인자를 찾을 수 있다.

Seurat 분석은 함수의 연속입니다

pbmc <- NormalizeData(pbmc)
pbmc <- FindVariableFeatures(pbmc, nfeatures = 2000)
pbmc <- ScaleData(pbmc)
pbmc <- RunPCA(pbmc)

각 줄은 입력 객체 → 함수 → 반환 객체 구조를 가집니다.

함수는 입력을 받아 작업하고 값을 반환합니다

입력

pbmc

함수

NormalizeData()

반환값

정규화 결과가 추가된 객체

함수가 화면에 무엇을 출력하는지와 실제로 무엇을 반환하는지는 다를 수 있습니다.

괄호 안의 값이 인자입니다

FindClusters(
  object = pbmc,
  resolution = 0.5
)
  • object, resolution: 인자 이름
  • pbmc, 0.5: 전달하는 값
  • 인자는 함수의 동작을 조절합니다.

위치 인자는 순서로 의미를 결정합니다

round(3.14159, 2)
round(x = 3.14159, digits = 2)

두 호출은 같은 결과를 냅니다. 이름을 쓰면 코드의 의미가 더 분명해집니다.

이름 있는 인자는 순서를 바꿀 수 있습니다

round(digits = 2, x = 3.14159)

Seurat 함수에서는 중요한 설정을 이름과 함께 쓰는 것을 권장합니다.

FindClusters(pbmc, resolution = 0.5)

기본값은 생략했을 때 사용됩니다

NormalizeData(pbmc)

공식 튜토리얼의 더 명시적인 표현:

NormalizeData(
  pbmc,
  normalization.method = "LogNormalize",
  scale.factor = 10000
)

기본값도 분석 기록에서 중요하면 명시할 수 있습니다.

도움말에서 함수 계약을 읽습니다

?NormalizeData
args(NormalizeData)

확인할 내용:

  • 어떤 클래스의 입력을 받는가?
  • 필수 인자는 무엇인가?
  • 기본값은 무엇인가?
  • 어떤 객체를 반환하는가?

반환값을 저장하지 않으면 다음 단계에 남지 않습니다

# 결과를 화면에 계산하지만 pbmc에는 다시 저장하지 않음
NormalizeData(pbmc)

# 반환된 객체를 pbmc에 다시 저장
pbmc <- NormalizeData(pbmc)

Seurat 튜토리얼에서 같은 객체 이름에 다시 할당하는 이유입니다.

어떤 함수는 요약값을 반환합니다

nrow(pbmc)
ncol(pbmc)
VariableFeatures(pbmc)

이 함수들은 Seurat 객체에 분석 단계를 추가하기보다 정보 또는 값을 가져옵니다.

top10 <- head(VariableFeatures(pbmc), 10)

함수 호출은 안쪽부터 계산됩니다

top10 <- head(VariableFeatures(pbmc), 10)

실행 순서:

  1. VariableFeatures(pbmc)
  2. 반환된 벡터를 head(..., 10)에 전달
  3. 결과를 top10에 저장

긴 함수 호출은 여러 줄로 정렬합니다

pbmc <- CreateSeuratObject(
  counts = pbmc.data,
  project = "pbmc3k",
  min.cells = 3,
  min.features = 200
)
  • 쉼표 뒤에서 줄을 바꿉니다.
  • 같은 수준의 인자를 같은 들여쓰기로 맞춥니다.
  • 닫는 괄호는 함수 시작 위치에 맞춥니다.

파이프는 왼쪽 결과를 다음 함수로 전달합니다

pbmc.markers %>%
  group_by(cluster) %>%
  filter(avg_log2FC > 1) %>%
  slice_head(n = 10)

위에서 아래로 “그 다음 무엇을 하는가”를 읽습니다.

기본 R의 |>도 같은 방향으로 데이터를 전달합니다.

함수 이름이 겹치면 패키지를 명시합니다

dplyr::filter(pbmc.markers, avg_log2FC > 1)
stats::filter(expression_values)

같은 filter()라도 패키지에 따라 완전히 다른 작업을 합니다.

오류 메시지에서 함수와 인자를 찾습니다

FindClusters(pbmc, resoluton = 0.5)

철자가 틀린 인자는 사용되지 않거나 오류를 발생시킵니다.

확인 순서:

  1. 오류를 낸 함수 이름
  2. 잘못되었다고 표시된 인자
  3. args()와 도움말의 정확한 철자
  4. 전달한 값의 자료형과 범위

간단한 사용자 정의 함수를 만듭니다

percent <- function(part, total) {
  part / total * 100
}

percent(25, 200)
  • 함수 이름: percent
  • 인자: part, total
  • 마지막 계산값이 반환값

10회차에서 분석용 함수를 더 자세히 만듭니다.

실습 1: 함수 호출 해체하기

pbmc <- FindVariableFeatures(
  pbmc,
  selection.method = "vst",
  nfeatures = 2000
)

다음을 표시하세요.

  1. 함수 이름
  2. 입력 객체
  3. 이름 있는 인자 두 개
  4. 반환값이 저장되는 객체

실습 2: 도움말로 코드 수정하기

DimPlot(pbmc, reduction = "umap")

?DimPlot을 확인하여 다음을 수행하세요.

  1. cluster 라벨 표시
  2. 점 크기를 0.7로 변경
  3. 범례를 유지한 결과와 제거한 결과 비교

오늘의 핵심

  • 함수는 입력과 인자를 받아 값을 반환합니다.
  • 이름 있는 인자는 코드의 의미를 명확하게 합니다.
  • 생략한 인자에는 기본값이 적용됩니다.
  • 중첩 호출은 가장 안쪽 함수부터 읽습니다.
  • 반환값을 객체에 저장해야 다음 분석 단계에서 사용할 수 있습니다.

  3. head(VariableFeatures(pbmc), 10)의 실행 순서는 무엇인가요?
