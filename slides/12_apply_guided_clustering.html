Zeisel 데이터로 Guided Clustering 완주하기

RDS 로드부터 cluster 해석과 결과 저장까지

12회차

오늘의 학습 목표

  1. 저장된 Zeisel Seurat 객체를 불러와 구조와 metadata를 점검할 수 있다.
  2. Guided Clustering의 QC부터 marker 탐색까지 같은 흐름으로 실행할 수 있다.
  3. 분석 파라미터를 바꾸고 cluster를 저자 제공 label과 비교·해석할 수 있다.

오늘은 새 데이터로 전체 흐름을 직접 완주합니다

1회차 12회차
PBMC 3K 코드를 따라 실행 Zeisel mouse brain 객체에 직접 적용

분석 순서는 같지만 데이터와 생물학적 맥락은 다릅니다.

QC 기준은 Zeisel 분포에서 다시 정합니다

PBMC 기준을 mouse brain 데이터에 그대로 복사하지 않습니다.

Mouse mitochondrial gene은 ^mt-로 찾습니다

sum(grepl("^mt-", rownames(brain)))

brain[["percent.mt"]] <- PercentageFeatureSet(
  brain,
  pattern = "^mt-"
)

PBMC tutorial의 human gene symbol은 보통 ^MT-를 사용합니다.

대소문자 하나가 QC 값을 완전히 바꿀 수 있으므로 먼저 feature 이름을 확인합니다.

임계값보다 분포를 먼저 봅니다

VlnPlot(
  brain,
  features = c(
    "nFeature_RNA",
    "nCount_RNA",
    "percent.mt"
  ),
  ncol = 3,
  pt.size = 0
)
summary(brain$nFeature_RNA)
summary(brain$percent.mt)

낮은 feature 수, 지나치게 높은 feature 수, 높은 mitochondrial 비율을 확인합니다.

이번 실습의 QC 값은 시작점이지 정답이 아닙니다

min_features <- 1000
max_features <- 7500
max_percent_mt <- 25

keep <- brain$nFeature_RNA > min_features &
  brain$nFeature_RNA < max_features &
  brain$percent.mt < max_percent_mt

table(keep)

현재 객체에서는 이 기준으로 약 2,900개 세포가 남습니다.

필터 전후 세포 수를 반드시 기록합니다

cells_before_qc <- ncol(brain)

brain <- subset(
  brain,
  subset = nFeature_RNA > min_features &
    nFeature_RNA < max_features &
    percent.mt < max_percent_mt
)

c(before = cells_before_qc,
  after = ncol(brain))

subset()은 조건을 만족하는 세포로 구성된 새 Seurat 객체를 반환합니다.

전처리는 객체에 새 분석 결과를 추가합니다

정규화 → 고변이 유전자 → scaling 순서를 추적합니다.

NormalizeData()는 normalized data layer를 만듭니다

brain <- NormalizeData(
  brain,
  normalization.method = "LogNormalize",
  scale.factor = 10000
)

Layers(brain[["RNA"]])
입력 핵심 인자 객체 변화
raw counts method, scale factor normalized data layer 추가

고변이 유전자 2,000개가 PCA의 입력이 됩니다

brain <- FindVariableFeatures(
  brain,
  selection.method = "vst",
  nfeatures = 2000
)

top10_variable <- head(
  VariableFeatures(brain),
  10
)
length(VariableFeatures(brain))
VariableFeaturePlot(brain)

모든 gene이 아니라 cell 간 차이를 잘 드러내는 gene에 집중합니다.

scaling 후 PCA로 주요 변화 축을 찾습니다

brain <- ScaleData(
  brain,
  features = VariableFeatures(brain)
)

brain <- RunPCA(
  brain,
  features = VariableFeatures(brain),
  npcs = 30
)
Reductions(brain)
print(brain[["pca"]], dims = 1:5, nfeatures = 5)

PC 수는 다음 단계의 입력 파라미터입니다

ElbowPlot(brain, ndims = 30)

DimHeatmap(
  brain,
  dims = 1:10,
  cells = 500,
  balanced = TRUE
)

이번 실습의 공통 시작값:

dims_to_use <- 1:20

Elbow와 PC loading을 근거로 다른 값을 시험할 수 있습니다.

같은 PC로 graph clustering과 UMAP을 계산합니다

dims_to_use가 두 분석의 공통 입력입니다.

이웃 그래프를 만든 뒤 cluster를 찾습니다

brain <- FindNeighbors(
  brain,
  dims = dims_to_use
)

brain <- FindClusters(
  brain,
  resolution = 0.5,
  random.seed = 42
)

table(Idents(brain))

resolution이 커지면 일반적으로 cluster가 더 세분화됩니다.

UMAP은 cluster를 계산하지 않습니다

brain <- RunUMAP(
  brain,
  dims = dims_to_use,
  seed.use = 42
)

DimPlot(
  brain,
  reduction = "umap",
  label = TRUE,
  repel = TRUE
)

UMAP은 선택한 PC 공간을 2차원으로 표현하는 시각화입니다.

cluster와 reference label을 같은 좌표에서 비교합니다

cluster_umap <- DimPlot(
  brain,
  group.by = "seurat_clusters",
  label = TRUE
)

reference_umap <- DimPlot(
  brain,
  group.by = "level1class",
  label = TRUE
)

cluster_umap + reference_umap

좌표는 같고 점의 색을 정하는 변수만 다릅니다.

교차표는 cluster의 섞임을 수치로 보여줍니다

cluster_reference <- table(
  cluster = Idents(brain),
  reference = brain$level1class
)

cluster_reference
round(
  prop.table(cluster_reference, margin = 1),
  2
)

cluster마다 어떤 reference class가 많은지, 여러 class가 섞였는지 확인합니다.

Marker를 근거로 cluster를 해석합니다

label을 붙이기 전에 differential expression 결과를 확인합니다.

모든 cluster의 positive marker를 계산합니다

markers <- FindAllMarkers(
  brain,
  only.pos = TRUE,
  min.pct = 0.25,
  logfc.threshold = 0.25,
  max.cells.per.ident = 100,
  random.seed = 42
)

max.cells.per.ident = 100은 60분 실습을 위한 subsampling입니다. 최종 연구 분석에서는 전체 세포 결과와 비교합니다.

top_markers <- markers |>
  filter(p_val_adj < 0.05) |>
  group_by(cluster) |>
  slice_max(avg_log2FC, n = 5) |>
  ungroup()

FindAllMarkers()의 비교 단위는 현재 Idents(brain)입니다.

Mouse brain marker panel로 교차 확인합니다

marker_panel <- c(
  "Gad1", "Slc17a7", "Mbp",
  "Aldoc", "C1qa", "Pdgfrb"
)

DotPlot(
  brain,
  features = marker_panel
) + RotatedAxis()

한 marker 하나가 아니라 top marker, marker panel, reference label을 함께 봅니다.

DoHeatmap()에 넣을 top marker는 객체 복사본에서 ScaleData(features = ...)로 scaling합니다. 원래 brain의 scale.data는 유지합니다.

교육용 reference label은 해석을 검증하는 장치입니다

majority_labels <- as.data.frame(cluster_reference) |>
  filter(Freq > 0) |>
  group_by(cluster) |>
  slice_max(Freq, n = 1, with_ties = FALSE) |>
  ungroup()
new_ids <- setNames(
  as.character(majority_labels$reference),
  as.character(majority_labels$cluster)
)

brain <- RenameIdents(brain, new_ids)

실제 연구에서는 marker 근거와 외부 reference를 별도로 검토해야 합니다.

이제 Guided Clustering 코드를 새 데이터에 적용할 수 있습니다

  • readRDS()로 분석 출발점을 복원했습니다.
  • 분포를 보고 Zeisel 데이터의 QC 조건을 적용했습니다.
  • 정규화, PCA, graph clustering, UMAP을 같은 객체에 누적했습니다.
  • marker와 저자 제공 label을 함께 사용해 cluster를 검증했습니다.
  • 파라미터를 하나씩 바꾸고 결과를 저장했습니다.

분석을 실행했다는 사실보다, 각 값의 근거와 객체의 변화를 설명하는 것이 최종 목표입니다.
